# 웰컴 메시지 미발송 버그

## 증상

네이버 스마트스토어 챗봇 에이전트의 *웰컴 메시지*가 사용자가 처음 접속해도 *발송되지 않음*.

재접속 시 에러 로그에 외부 시스템 코드 (예: `42703` — DB 인터페이스가 알려지지 않은 칼럼을 만났을 때 내는 에러) 가 보이고, 그 이후 메시지가 끊긴다.

## 의도된 동작

- 첫 접속 → 웰컴 메시지 발송
- 당일 재접속 → 미발송
- 날짜 변경 후 재접속 → 재발송
- 다른 사용자 첫 접속 → 발송

## Acceptance criteria

- `daily_access_tracker.py`의 `is_first_access_today()`가 *정상 동작*하여 첫 접속 여부를 정확히 판별한다.
- `test_daily_access_tracker.py`가 통과한다.
- 회귀 없음: 기존 동작(재접속 시 미발송 등)이 유지된다.

## Out-of-scope

- 웰컴 메시지 *내용*(템플릿)은 변경하지 않는다.
- 다른 에이전트(naver-talktalk 등)의 동등 로직은 손대지 않는다.
