# 상담사 개입 시 봇 이중 응답 버그

## 증상

네이버 스마트스토어 채팅에서 상담사가 대화에 *개입한 이후에도* 봇이
사용자 메시지에 계속 응답하여 *이중 답변*이 발생함.

## 의도된 동작

- 상담사가 *주도권*을 보유한 세션 → 봇은 응답하지 않음
- 상담사가 대화에서 *나간(handover)* 시그널 발생 → 봇은 응답하지 않음
- 상담사가 *공식 절차 없이* 사용자에게 직접 답변을 입력하기 시작한
  경우 → 봇은 *주도권을 상담사에게 공식 이양*하고 응답하지 않음

## Acceptance criteria

- 상담사 주도권 상태 (상담사가 인수한 세션)에서 사용자 발화가 오면
  봇이 *응답하지 않고* HTTP 200을 반환한다.
- 핸드오버 완료/시작 이벤트가 오면 봇은 *응답하지 않고* 로그를 남긴
  뒤 HTTP 200을 반환한다.
- 상담사가 비공식적으로 사용자에게 답변을 입력한 정황(특정 sourceId
  + manager 닉네임)이 감지되면, 봇은 *대신* passThread를 한 번 발송해
  주도권을 공식 이양한다.
- `tests/test_welcome_message.py`의 신규 테스트가 통과한다.
- 회귀 없음: 일반 사용자 메시지에 대한 기존 응답 흐름이 깨지지 않는다.

## Out-of-scope

- 다른 에이전트 (naver-talktalk 등)의 동등 로직은 *손대지 않는다*.
- 상담사 화면(파트너센터) 측 변경은 *불가* — 봇 쪽 가드만으로 해결한다.
