# 상품 가격 견적 환각 버그

## 증상

스마트스토어 챗봇이 상품 가격을 답할 때 다음 3가지 결함이 동시에 발생함:

- **컨텍스트 오염** — 상품 정보 조회 도구가 raw 응답(약 80KB JSON)을 그대로
  LLM 에 넘기면서, 상품의 *과거 가격 캐시 필드*(현재가와 다른 값)가
  정상 단가와 함께 노출됨.
- **산수 환각** — 같은 답변 안에서 요약 총액과 옵션별 상세 합산 총액이
  서로 다른 값으로 등장 (예: 428,000원 vs 462,000원).
- **내부 필드 유출** — 옵션 차감금(음수) 같은 *사용자에게 보이면 안 되는*
  필드가 본문에 그대로 인용됨 (예: "아동 -4,100원").

실제 결제 금액과 LLM 답변이 *수만 원 단위로 어긋남* — 환불·재고지 비용
발생하는 운영 사고.

## 의도된 동작

- 상품 정보 조회 도구는 *정제된 한국어 텍스트* 만 LLM 에 전달하고,
  raw JSON 페이로드는 LLM 에 노출하지 않는다.
- 옵션별 단가는 단일 진실 소스 헬퍼가 산출하고, 가격 표현은 한 곳에서만
  계산되어 본문에 일관되게 인용된다.
- 총액(인원·수량 합산) 은 *LLM 산수에 맡기지 않고* 결정론 계산기를
  서버 사이드에서 호출해 응답한다.
- 차감금(음수) 같은 내부 필드는 본문에 노출되지 않는다.

## Acceptance criteria

- `agents/naver-smartstore/app/services/price_quote.py` 가 신설되어
  `compute_quote` 결정론 계산 함수가 정의된다 (옵션 키워드·수량 입력 →
  총액·문구 출력).
- `agents/naver-smartstore/app/services/product_context.py` 에
  `resolve_option_unit_prices` 헬퍼가 추가되어 옵션 단가 산출의 *단일
  진실 소스* 가 된다. 본문 빌더는 이 헬퍼를 소비한다.
- `agents/naver-smartstore/app/routers/product.py` 의 상품 정보 조회
  도구가 raw 페이로드 머지 분기를 *제거* 하고, 정제 텍스트만 반환한다.
- 같은 라우터에 `quote_price` 도구가 신규 등록되어 PRODUCT 의도 처리시
  LLM 이 견적 계산을 도구 호출로 위임할 수 있다.
- `agents/naver-smartstore/app/prompts/product.md` 에 `quote_price` 도구
  의무 사용과 단가 출처 한정 규칙이 명시된다.
- 사건 fixture 기반 회귀 테스트가 신설되어 *금지 수치 미노출* 과 *정상
  총액 일치* 를 잠근다 (테스트가 fix 검증의 일부).

## Out-of-scope

- 다른 에이전트(naver-talktalk, kakao 등) 의 가격 응답 흐름은 손대지
  않는다.
- 상품 정보 조회 도구의 voucher 안내 블록 OCR 누락은 별도 follow-up
  (이번 변경 범위 아님).
- 4개 이상의 옵션 그룹을 가진 상품은 별도 PR 에서 확장.
